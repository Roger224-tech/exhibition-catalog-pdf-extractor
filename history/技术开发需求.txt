一、项目名称
展会目录PDF自动提取工具（贸易商版）

二、项目背景
贸易商在展会上收集大量参展商提供的产品目录（PDF格式），这些目录包含产品图片、名称、型号、参数、规格等信息。传统人工整理效率低、易错漏。本工具旨在自动解析PDF目录，将产品信息结构化导出为Excel表格，并自动匹配对应图片，提升数据录入与分类效率。

三、核心功能需求
1. PDF文件批量导入
支持单文件或多文件（文件夹）导入

支持PDF格式版本：PDF 1.4 ~ 2.0

支持扫描件（图像型PDF）及原生电子版PDF（文字型PDF）

2. 产品信息自动识别与提取
需自动识别并提取以下常见字段（允许自定义扩展）：

字段示例	说明
产品编号 / 型号	如 “Model: X-100”
产品名称	如 “高精度数字万用表”
品牌 / 制造商	如 “Fluke”
规格参数	如 “测量范围：0-1000V”
简要描述	产品特点或用途
价格（若有）	支持货币符号识别
页码信息	该产品在PDF中的页码，便于回溯
识别逻辑：

文字型PDF：直接提取文本及布局位置

扫描件：内嵌OCR（支持中英文混排）

支持常见版式：表格型、图文混排型、产品卡片型

3. 图片自动截取与对齐
提取每个产品对应的主图（1张，如有副图可配置提取数量）

图片保存为独立文件（如jpg/png），并按规则命名：产品编号_原页码.jpg

在Excel中为每个产品行自动插入图片（缩略图）或填入图片文件路径
（建议提供选项：缩略图嵌入 或 路径引用）

4. Excel输出格式自动对齐
输出 .xlsx 文件

表头字段自动对齐：根据识别出的字段名映射到标准列（如“产品编号”列、“产品名称”列）

支持用户预设字段映射模板（如某展会目录列名不同，可手动校对一次后存储模板）

每一行对应一个独立产品，列对齐无错位

5. 智能字段匹配 （针对多变的目录结构）
自动判断PDF中的产品分界（如通过页码、分割线、标题样式变化）

对于一行多列的产品列表（例如每页有2x3个产品卡片），能正确拆分每个产品块

当字段缺失时，留空单元格并给出提示标记

6. 人工校对辅助界面（可选但强烈建议）
生成Excel前，提供一个预览界面：

左侧PDF原页面，右侧已提取的产品字段列表

支持拖拽修正错误的字段对齐、删除多余识别内容

支持手动添加漏识别的产品

7. 处理结果报告
生成提取日志：成功提取产品数、图片数、无法识别的页码列表

若有明显异常（如某页无产品图、字段完全缺失），高亮提示

四、非功能性需求
项目	要求
操作系统	Windows 10/11 (优先) ，可选macOS
运行方式	桌面客户端（.exe）或Web工具（私有部署）
处理速度	单份100页PDF处理时间 ≤ 3分钟（含OCR）
准确率要求	文本字段提取准确率 ≥ 90%（标准展会版式）
图片与产品匹配准确率 ≥ 95%
文件大小限制	单个PDF ≤ 200MB
数据隐私	所有处理在本地完成，不上传任何文件至云端
五、技术实现建议（供开发参考）
PDF解析库：PyMuPDF(fitz) 或 pdfplumber 用于文字型；Tesseract OCR + pdf2image 用于扫描件

图像处理：OpenCV 或 PIL 进行图片裁剪与增强

表格生成：openpyxl 或 pandas + xlsxwriter（支持图片嵌入）

字段对齐与拆分：基于规则引擎（正则+位置坐标） + 可训练的简易分类器（如基于版面分析）

GUI框架：PyQt / Tkinter （桌面版）或 Flask + 前端（Web版）

六、交付物清单
可执行程序（带图形界面）

用户操作手册（PDF格式）

示例输入PDF及对应输出Excel（不少于3种不同版式）

安装与配置说明（如需要Python环境则提供打包exe）

简单维护文档（字段映射模板的编辑方法）

七、可选增强功能（按需增加预算）
自动识别多语言（英/中/日/韩）

支持输出到数据库（SQLite, MySQL）

支持将图片上传至指定云端存储并生成链接列

与贸易商ERP系统对接（提供API接口）
