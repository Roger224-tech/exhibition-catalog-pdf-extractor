展会目录AI智能提取工具（汽配版）—— 技术开发需求书 V2.0
核心定位：以AI自动识别为主，以人工预览校验为辅，为汽配贸易商量身定制的PDF数据提取工具。

一、项目背景与目标
1.1 业务痛点
汽配贸易商在展会上收集大量不同供应商、版式各异的产品目录PDF

目录中包含车型适配、OE号、产品图片等关键信息，但排版复杂（无统一表格，多采用自由卡片式布局）

现有工具要么识别率低，要么无法关联图片，要么缺乏人工修正机制

完全自动化AI工具准确率有限，完全人工整理效率低下

1.2 项目目标
开发一款 “AI识别 + 人工校验” 模式的桌面工具：

AI优先识别：自动提取产品信息及对应图片

人工预览修正：提供直观界面，允许用户修改、删除、补充识别结果

最终输出：生成符合用户期望的Excel表格（字段对齐、图片关联）

二、核心功能需求
2.1 PDF导入模块
需求项	说明
文件格式	支持 PDF（版本1.4~2.0），包括文字型及扫描件（图像型）
批量导入	支持单文件或文件夹批量导入
文件大小	单个文件 ≤ 200MB
语言支持	中英文混排（汽配术语：OE号、车型、品牌等）
2.2 AI智能识别模块（核心）
2.2.1 字段提取
自动识别并提取以下汽配相关字段（用户可自定义扩展）：

字段	示例	说明
产品编号 / OE号	BMW-345267、12345-67890	配件唯一编号
产品名称	刹车片、前保险杠	配件名称
品牌 / 制造商	Bosch、ZF	品牌信息
车型适配	宝马3系 E90 2005-2012	适配车型及年份
规格参数	尺寸：150mm、材质：铸铁	技术参数
简要描述	原厂品质，耐磨性强	产品描述
价格（若有）	$45.99	支持多货币
原厂参考号	L321-9876	交叉引用编号
每包数量	1套/盒	包装信息
2.2.2 图片提取与关联
自动识别每个产品对应的主图（如有副图，可配置提取数量）

图片裁切并保存为独立文件（产品编号_页码_序号.jpg）

AI自动判断图片与产品的对应关系（如距离最近、框内包含等）

2.2.3 AI模型要求
支持文字型PDF直接提取文本位置及内容

支持扫描件PDF内置OCR（中英文混排识别）

支持无表格、多卡片、图文混排等复杂排版

每个提取字段附带置信度评分（0-100%），供后续人工校验参考

2.3 人工预览校验模块 ★核心亮点★
2.3.1 界面布局
采用左右分屏设计：

区域	内容	交互功能
左侧	PDF原始页面预览	支持缩放、翻页；选中右侧某条数据时，左侧高亮显示该数据在原PDF中的位置
右侧	AI提取的结构化数据表格	展示当前页码/当前产品的所有字段；每行代表一个提取结果
2.3.2 校验操作（用户可执行的修改）
操作	说明
编辑字段内容	双击单元格，直接修改文本内容
删除字段	删除错误或无关的字段行
新增字段	手动添加AI漏识别的字段（用户先在左侧框选PDF区域，再填写字段名称和值）
合并产品	将本页多个识别块合并为一个产品（如跨行描述）
拆分产品	将一个识别块拆分为多个独立产品（如AI将两产品误识别为一个）
标记无效	标记某条数据为“非产品内容”（如页眉页脚、广告），供AI学习忽略
2.3.3 AI置信度辅助
右侧表格中，置信度低于阈值（如70%）的单元格高亮显示（如黄色背景）

用户修改过的字段，保存修改记录，可选择性用于后续模型微调

2.3.4 批量操作
支持批量删除某页所有无效字段

支持批量复制某字段值到同页其他产品

支持模板应用：将某一页的修正规则快速应用到相同版式的其他页面

2.4 Excel导出模块
2.4.1 输出格式
文件格式：.xlsx

表头字段：根据用户预设的字段映射表自动对齐

每一行 = 一个独立产品

2.4.2 图片处理（二选一，可配置）
选项	说明
缩略图嵌入	在Excel单元格内直接显示产品图片缩略图
路径引用	在单元格内填写图片文件本地路径，图片单独保存在文件夹中
2.4.3 字段模板
用户可保存当前字段结构为模板

下次处理同类目录时，一键应用该模板（自动对齐列、忽略无关字段）

支持模板的导入/导出

2.5 处理报告模块
生成处理日志：总文件数、总产品数、提取图片数

标记无法识别或识别异常的页码（无产品、无图片、字段全部低置信度）

统计人工修改次数（可选）

三、非功能性需求
项目	要求
操作系统	Windows 10/11（优先），可选macOS
运行方式	桌面客户端（.exe或.dmg），无需联网（OCR引擎可本地）
处理速度	单份100页PDF（含OCR）≤ 5分钟；预览界面操作流畅无卡顿
识别准确率	AI初次识别：文本字段 ≥ 85%；图片-产品匹配 ≥ 90%
（经人工校验后，最终输出准确率目标100%）
数据隐私	所有处理均在本地完成，不上传任何文件至云端
易用性	无需编程知识；界面简洁，操作引导清晰
四、技术实现建议（供开发参考）
模块	推荐技术栈
PDF解析	PyMuPDF（文字型）+ pdf2image + Tesseract 5.x（扫描件OCR）
AI识别模型	选用开源版面分析模型：LayoutLMv3 或 Table Transformer，在汽配目录样本上微调；或用 MinerU / Chunkr 作为预处理引擎
图像处理	OpenCV（图片裁切、增强）
图片-产品关联	基于空间距离 + 版面聚类算法
前端界面	PyQt6 或 Tkinter（桌面原生） + pdf.js（PDF预览组件）
Excel导出	openpyxl（支持图片嵌入）或 pandas + xlsxwriter
模板管理	JSON / SQLite 本地存储
五、交付物清单
可执行程序（带图形界面，绿色免安装版或安装包）

用户操作手册（PDF格式，含图文教程）

示例数据包：5份不同版式的汽配目录PDF + 对应的输出Excel（展示最大能力）

字段模板示例：3个典型供应商的预置模板

快速上手指南（1页纸）

维护文档：如何新增字段、如何导出训练数据、模板备份恢复方法

六、可选扩展功能（视预算与时间）
功能	说明
模型持续学习	将用户人工修改的数据导出，用于后续模型微调，提升自动化准确率
多语言支持	支持日文、韩文、德文汽配目录（常见进口配件）
数据库直接写入	支持将提取结果直接写入用户指定的 SQLite / MySQL 数据库
云端协同	支持团队共享模板、校验记录（需登录及私有云部署）
VIN码智能解析	自动识别VIN码并解析出车型、年款、发动机等信息
七、验收标准
编号	验收项	标准
1	PDF导入	支持100页、200MB以内的PDF加载，耗时 ≤ 10秒
2	AI识别（文字型）	10份不同版式汽配目录，字段提取准确率 ≥ 85%
3	AI识别（扫描件）	5份扫描件（300dpi），中英文OCR可读，字段提取准确率 ≥ 75%
4	图片提取	图片裁切完整，与对应产品关联准确率 ≥ 90%
5	预览校验	左右分屏、高亮定位、增删改查功能全部可用，无崩溃
6	Excel导出	根据模板对齐字段，图片正确嵌入或引用，无数据错位
7	处理报告	日志完整，异常页码标记清晰
八、项目风险与应对
风险	可能性	应对措施
超复杂排版AI识别率过低	中	强化预览校验功能，让用户快速修正；提供“模板复制”减少重复工作
OCR识别中文误差大	中	采用Tesseract + 中文训练数据，或集成商业OCR引擎（如百度离线SDK）作为备选
图片-产品关联错误	中	在预览界面允许用户“重新关联”：右侧选中产品，左侧手动框选正确图片
开发周期超预期	中	分阶段交付：V1.0实现文字型PDF + 预览校验；V2.0加入扫描件OCR + 模型微调
